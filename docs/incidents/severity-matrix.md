# Incident Severity Matrix

## Severity Definitions

1. **Sev1**
   - Widespread production outage or critical billing/security impact.
   - Acknowledge within 5 minutes.
2. **Sev2**
   - Major feature degradation with active user impact.
   - Acknowledge within 10 minutes.
3. **Sev3**
   - Limited degradation or single-surface issue.
   - Acknowledge within 30 minutes.
4. **Sev4**
   - Minor issue with workaround.
   - Handle in normal backlog flow.

## Mandatory Roles (Sev1/Sev2)

1. Incident commander.
2. Comms lead.
3. Scribe.
4. Technical lead.
